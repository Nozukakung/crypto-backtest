## 1. ภาพรวมกลยุทธ์

### ชื่อกลยุทธ์
**Mean Reversion + DCA โดยใช้ RSI เป็น Signal**
❌ ยืนยันชัดว่า **ไม่ใช่ Martingale**

### แนวคิดหลัก
- ใช้ RSI(14) บน TF 1m หาโซน Overbought/Oversold
- ใช้ Candle Pattern กรอง signal เพิ่ม
- เข้าไม้แบบ **Maker limit order** ทุกไม้ (ประหยัดค่า fee)
- ใช้ DCA ลดต้นทุนเฉลี่ย (EP) แล้วปิดที่ **BEP + 0.2%**
- **Win rate 100%** (เพราะไม่ตัดขาดทุน ทยอยถัวจนราคากลับ)

### สภาพแวดล้อม
- Timeframe: 1 นาที (TF 1m) ทั้งหมด
- คู่เงิน: BTC, ETH, SOL (คาดการณ์)
- Fee: Maker (limit) ถูกกว่า Taker ~0.025%
- รันแบบ: บอทอัตโนมัติเต็มตัว ใช้ WebSocket ดึงราคา real-time

## 2. เงื่อนไขเปิด Position (Signal Engine)

### 📈 เงื่อนไข Long (ต้องเข้าครบทุกข้อ)

1. **RSI(14) บน 1m < 40** (โซน Oversold)
2. **แท่งล่าสุดไม่ปิดที่ Low** (มีแรงเด้ง)
3. **ไม่มี Pattern ต่อไปนี้:**
   - ไม่มีแท่งเขียวปิดสูงติดกัน 5 แท่ง
   - ไม่มี Pattern "แตะ → แตะ → แตะ" (2-3 แท่ง)
4. **ราคา limit order = ราคา real-time − 0.05%**

### 📉 เงื่อนไข Short (ต้องเข้าครบทุกข้อ)

1. **RSI(14) บน 1m > 60** (โซน Overbought)
2. **ไม่เปิดที่ราคา High**
3. **ไม่มี Pattern "แตะ → เชี่ยว → เชี่ยว"**
4. **ราคา limit order = ราคา real-time + 0.05%**

### 🕯️ ระบบตรวจจับ Candle Pattern
- บอทรับข้อมูล OHLC มาดราฟแท่งเทียนเอง
- เช็คว่า ณ แท่งนี้คือ Pattern อะไร (ตามตำรา)
- ทำนายว่าแท่งต่อไปจะขึ้นหรือลง
- **นี่คือ "ไม้ตาย" ที่เจ้าตัวบอกว่าเลียนแบบยากสุด**

## 3. Order Engine (ระบบวางออเดอร์)

### กฎหลัก
- **ทุกออเดอร์ต้องเป็น Maker** (limit order)
- เพื่อได้ค่า fee ต่ำสุด
- ราคาวาง = ราคา real-time ± 0.05% (ให้เป็น Maker แน่ๆ)
- ถ้าเป็น Taker จะโดน fee เพิ่ม 0.025%

### กลไก Refresh Order
- ทุก **1 นาที** เช็คออเดอร์ที่ยังไม่ match
- ถ้าราคาขยับจนออเดอร์ไม่สมเหตุผล:
  - **ยกเลิกออเดอร์เก่า**
  - **คำนวณราคาวางใหม่** ตามราคา real-time
  - **วางออเดอร์ใหม่**
- Loop นี้ทำซ้ำ: คิด → วาง → เช็ค → คิด → วาง ทุกๆ นาที

## 4. Position Manager (จัดการสถานะ)

### พารามิเตอร์หลัก
| พารามิเตอร์ | ค่า |
|-------------|-----|
| Take Profit (TP) | BEP + 0.2% |
| DCA Trigger | ราคาต่ำกว่า BEP 0.3% |
| Max Position Size | $50,000/คู่ |
| Long timeout รีเฟรช | 30 นาที |
| Short timeout ออก | 120 นาที |

### นิยาม BEP (Break Even Point)
BEP = ราคาเปิด + ค่าใช้จ่ายทั้งหมด (fee + funding + PnL ติดลบจริง)
- บอทคำนวณ BEP จริงของแต่ละ position อัตโนมัติ
- ตั้งราคาปิด = BEP + 0.2% (การันตีกำไร 0.2%)

### กลไก DCA
- ราคาต่ำกว่า BEP 0.3% → **เพิ่ม position ขนาดเท่าเดิม**
- DCA แล้ว EP (ต้นทุนเฉลี่ย) ลดลง
- ยิ่งซื้อๆ ขายๆ → EP ยิ่งลดเข้าใกล้ BEP
- สุดท้ายราคากลับมา → ปิดได้เอง → กำไร
- **ทุกไม้ขนาดเท่ากัน** (ไม่เบิ้ล ไม่ลด)

### จัดการ Long timeout (เช็คทุก 5 นาที)
- ถ้า Long ค้างเกิน **30 นาที**:
  - ยกเลิกออเดอร์ทั้งหมด
  - วางใหม่ที่ **BEP + 0.2%**
  - รอ match

### จัดการ Short timeout (เช็คทุก 5 นาที)
- ถ้า Short ค้างเกิน **120 นาที**:
  - ส่ง signal ไป system (คาดว่าบังคับออก)

## 5. การควบคุมความเสี่ยง

### Max Position Size
- แต่ละคู่สูงสุด **$50,000**
- เป็น "position เจ้ามือระดับสูงสุด"
- ปกติไม่เปิดถึงขนาดนี้ แต่เป็นเพดานป้องกัน

### ไม่ตั้ง Stop Loss
- Win rate เคลม **100%**
- Logic: DCA ถัวจนต้นทุนลด ราคากลับมาแน่นอน
- **ความเสี่ยงไม่ใช่พอร์ตแตก** แต่คือ **"position ค้าง ติดดอยนาน กว่าจะกลับมากำไร"**

### ต่างจาก Martingale อย่างไร
| | Martingale | กลยุทธ์นี้ |
|---|---|---|
| Signal | ไม่มี เปิดมั่ว | RSI + Pattern |
| เพิ่มไม้ | เบิ้ลขนาด (exponential) | ขนาดคงที่ ไม่เบิ้ล |
| เพดาน | ไม่มี | มี Max $50K |
| เสี่ยงแตกพอร์ต | สูงมาก | ต่ำกว่า (ไม้คงที่) |

## 7. สถาปัตยกรรมเทคนิค

### การดึงข้อมูล
- **WebSocket** (ไม่ใช่ REST API) — ต้องได้ราคา real-time
- REST API ช้าไป
- ใช้ Connection Pool จัดการ
- นับ rate ต่อวินาที กัน Rate Limit

### Loop การทำงาน (ทุกนาที)
```
ทุก 1 นาที:
1. ดึง OHLC 1m ล่าสุดจาก WebSocket
2. คำนวณ RSI(14)
3. เช็ค Candle Pattern
4. เช็คเงื่อนไขเปิด position
5. ถ้าเข้าเงื่อนไข → วาง limit order (Maker)
6. เช็คออเดอร์ค้าง ถ้าราคาขยับ → ยกเลิก วางใหม่
7. เช็คทุก position:
   - Long > 30 นาที → วางใหม่ที่ BEP + 0.2%
   - Short > 120 นาที → ออก
   - ราคาต่ำกว่า BEP 0.3% → DCA
8. เช็ค Max Cap ($50K)
```

## 8. สิ่งที่ต้องทำ Backtest ให้แม่น

### Logic ที่ต้องเลียนแบบเป๊ะ
1. **RSI(14)** — ใช้ Wilder's RSI มาตรฐาน
2. **Candle Pattern** — ส่วนยากสุด ต้อง:
   - จับ Pattern 2-3 แท่ง
   - เช็คการปิดสูง/ต่ำติดกัน
   - Pattern matching
3. **Maker limit** — ราคา = real-time ± 0.05%
4. **BEP dynamic** — รวม fee + funding
5. **DCA** — ไม้คงที่ trigger ที่ต่ำกว่า BEP 0.3%
6. **Timeout** — Long 30 นาที, Short 120 นาที
7. **Refresh order** — ยกเลิกวางใหม่ทุกนาที

## 9. สูตรสำคัญ

### คำนวณ BEP
```
BEP = Σ(ราคาเปิด × ขนาดไม้) / Σ(ขนาดไม้) + fee รวม + funding รวม
```

### ราคา Take Profit
```
TP = BEP × (1 + 0.2/100) = BEP × 1.002
```

### Trigger DCA
```
ถ้า ราคาปัจจุบัน < BEP × (1 − 0.3/100) = BEP × 0.997:
    → เพิ่มไม้ (ขนาดเท่าเดิม)
    → อัปเดต EP
```

### ราคา Maker limit
```
LONG:  OrderPrice = ราคา × (1 − 0.05/100) = ราคา × 0.9995
SHORT: OrderPrice = ราคา × (1 + 0.05/100) = ราคา × 1.0005
```
